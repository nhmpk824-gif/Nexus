## Voice / TTS

- Fix tool-call replies dropping from TTS: `finish()` was closing the streaming controller after the first LLM round, so post-tool text never reached the player. Split `flushPending()` / `finish()` and call `finish()` only at the end of the whole agent turn.
- Empty PCM streams from the remote TTS provider now surface as an error event instead of silently "finishing without playback".
- First-audio watchdog: fail the streaming controller and fall back to direct speech if no chunk arrives within 6 s of `ttsStreamStart`.
- Markdown `**bold**` / `__bold__` / `` `code` `` stripped from the text handed to TTS — no more "star-star" or punctuation stumbles.
- Post-TTS listen window: 4.8 s → **3 s** (`API_RECORDING_MAX_IDLE_MS`).
- Wake-word ACK delay 120 → **30 ms**; wake cooldown 1500 → **500 ms** so repeat invocations don't need 1.5 s between them.
- VAD default sensitivity raised to `'high'` + high-preset threshold 0.22 → **0.18** to catch the first post-wake syllable on weak mics.
- `SherpaKws` logs an explicit "mic volume too low" warning after two consecutive low-peak windows so low gain is obvious, not mysterious.
- `VoiceSessionMachine` emits log-effect warnings on illegal `stt:finalizing` / `wake:detected` / `wake:debounced` transitions.
- TTS dispatch + mid-session voice-swap logs added so provider / timbre drift is diagnosable on first repro.

## Chat / Agent

- **Anthropic prompt caching** wired on system + tools prefix; current datetime moved out of the cached system prompt into a per-turn `<system-reminder>` block on the last user message so the prefix stays byte-stable.
- Search query framing: `stripSearchCommandFraming` shared between `extractSearchQuery` and `rewriteSearchQuery` so "那你能帮我找一下 X 的歌词吗" no longer reaches Tavily as garbage.
- `summarizeOlderMessages` now uses current `AppSettings` instead of empty provider fields (fixed `Failed to parse URL from /chat/completions` error every compaction pass).
- Error bubble no longer tears the chat list layout (CSS `min-width` + `word-break` + 500-char system-message cap).
- Autonomy broadcast dedupe: label-free key + 10 min window, so 【早报】X and 【自主】X collapse instead of both firing.
- **`vault:store-many` IPC**: `Object.keys(validated)`, not `validated.map` — this was the real reason the settings save button appeared dead.
